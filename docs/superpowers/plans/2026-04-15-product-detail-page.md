# Product Detail Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the Astro-first product detail page with a masonry desktop gallery, sticky detail panel, and mobile shadcn carousel using mock data only.

**Architecture:** Keep almost all of the page in Astro so the storefront stays lightweight. Use one React island only for the mobile carousel, and keep all mock content in `apps/web/src/lib/data.ts` so the route file stays a thin composition layer. The page should ship as a single mock slug now, but the route and data shape should stay easy to extend later.

**Tech Stack:** Astro, React island, shadcn/ui carousel, Tailwind classes, TypeScript, Ultracite, Prettier.

interface Props {
  images: ProductDetailGalleryImage[]
}

export function ProductGalleryMobileCarousel({ images }: Props) {
  return (
    <Carousel>
      <CarouselContent>
        {images.map((image) => (
          <CarouselItem key={image.caption}>
            ...
          </CarouselItem>
        ))}
      </CarouselContent>
    </Carousel>
  )
}
```

Keep the island focused on image slides and captions. Do not move the product detail copy into React.

```
